Literature workflow

Abstract screening

Deciding which records to keep from their titles and abstracts is already done well by free, dedicated tools, so this toolkit recommends the best one for your situation rather than bundling a weaker screener. Build and de-duplicate your search first, then screen in one of the tools below. Your records export as RIS or CSV, which all three import.

Which one should I use?

  1. No setup, any device: Catchii. Nothing to install, no account.
  2. Most powerful, if you can install software: ASReview. Machine-learning prioritisation for large sets.
  3. Two reviewers working together: Rayyan. Free tier, account required.
Recommended for most people

Catchii →

FreeNo accountNo installWorks offline
Runs in a browser on a phone, tablet, or laptop, and covers de-duplication, screening, and the PRISMA flow. The closest fit if you have limited bandwidth or no rights to install software.

Most powerful, needs a local install

ASReview LAB →

FreeOpen sourceActive learning
Validated machine-learning screening that ranks records by likely relevance and saves time on large sets. Needs Python or Docker installed. If you use it, cite van de Schoot et al. (2021), Nature Machine Intelligence 3:125–133, doi:10.1038/s42256-020-00287-7.

For collaborative screening

Rayyan →

Free tierAccount required
Web-based screening with two-reviewer blind mode and conflict resolution. The free tier covers screening; automated de-duplication and some features are paid.

Export your de-duplicated records as RIS or CSV with the de-duplicator, then import that file into the tool you choose.
