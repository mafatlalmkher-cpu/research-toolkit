About

Free tools, for whoever needs them

The Research Toolkit is a set of small, free, open-source tools for the research workflow, built as companions to QueryBee. The aim is plain: make a few genuinely useful tools that anyone can use, regardless of budget or institution.

Who this is for

Students, early-career researchers, and scientists in low-resource settings often cannot reach the paid software that makes parts of research easier. These tools close a few of those gaps. They run in any modern browser, need no account or licence, and keep your data on your own computer. A slow connection or a shared lab machine is enough.

How it is built

Each tool is a single HTML file with no build step and no dependency to install. The bench calculators run entirely on your device. The reference formatter and QueryBee use only free public services (Crossref, PubMed, and OpenAlex) to look up details, and store nothing. Because each tool is one file, you can download it, email it, archive it, or host it yourself.

Cite and reuse

Everything is released under the MIT licence, so you are free to use, copy, change, translate, and redistribute it, including for teaching. If the tools help your work, a citation to QueryBee is appreciated:

Kher, M. M. QueryBee: a Boolean query builder for literature search. Zenodo. https://doi.org/10.5281/zenodo.20733036

Host your own copy

You do not need permission, and you do not need a server. To put the whole site online for free:

  1. Create a free GitHub account, and a new repository.
  2. Upload these files, keeping the folder structure (the tool files, the assets folder, and the posts folder).
  3. In the repository settings, turn on GitHub Pages for the main branch.
  4. Your copy goes live at a public address within a minute or two.

Netlify and Cloudflare Pages work the same way by dragging the folder onto their dashboard. For a single tool, you do not even need that: the file works on its own from your computer.

Feedback

These tools are a work in progress and an effectiveness evaluation of QueryBee is underway. Corrections and suggestions are welcome through the project’s code repository.
