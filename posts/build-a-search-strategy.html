How to build a literature search that does not miss papers

A reliable search comes from structure, not from typing keywords into one database and hoping.

20 June 2026 · 2 min read

A search that misses relevant papers undermines everything built on top of it. The fix is not a longer keyword list but a clear structure, applied the same way in more than one database. Four steps get you most of the way there.

1. Start with concepts, not keywords

Break your question into two to four concepts. A question about eliminating a grapevine virus by tissue culture has three: the virus, the elimination method, and the host. Each concept is a separate idea that a relevant paper must mention in some form. Writing the concepts down first stops the search from drifting into a single bag of loosely related words.

2. Find the synonyms for each concept

Every concept hides behind several words: spelling variants, abbreviations, older names, and controlled-vocabulary terms such as MeSH. Authors will have used all of them. Pull the synonyms from the literature rather than from memory, because the terms you do not use are exactly the ones that cause misses.

3. Combine the terms with Boolean logic

Join synonyms within a concept with OR, and join the concepts with AND. Put each concept in its own field code and wrap it in parentheses so the database cannot regroup your logic. The result is one fully parenthesised string per database, built the same way every time.

4. Search more than one database, then de-duplicate and screen

No single database covers every journal. Run the same logic in Scopus, Web of Science, PubMed, and a database specific to your field, then combine the exports. You will collect the same paper several times, so de-duplicate before you start reading, and screen what is left in a structured, recorded way.
